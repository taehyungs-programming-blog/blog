## 사전지식 DXGI(DirectX Graphics Infrastructure)

**DXGI (DirectX Graphics Infrastructure)**는 DirectX 10부터 도입된 저수준 그래픽 인프라로, Direct3D와 하드웨어 사이의 공통 프레임워크를 제공

### 이게 왜 필요할까?

#### 근본적 이유

* DirectX 9까지는 D3D가 모든 걸 다 했어요. 디바이스 생성부터 화면 출력, 전체화면 전환까지 전부 D3D API 안에 있었죠. 그런데 이게 문제가 많았습니다:
* 중복 코드: D3D9, D3D10, D3D11... 각 버전마다 같은 기능(화면 출력, 모니터 열거 등)을 계속 다시 구현해야 했어요
* API 간 호환성 없음: D3D9과 D3D10을 같이 쓰려면 거의 불가능했죠

#### 관심사의 분리

* DXGI: "하드웨어와 OS 레벨" - GPU 찾기, 모니터 관리, 화면 출력
* D3D: "렌더링 API" - 그래픽스 파이프라인, 셰이더, 리소스
* 이렇게 나누니까 각자 역할이 명확해지고 코드도 깔끔해졌어요.

### DXGI의 역할

- **하드웨어 추상화**: GPU, 모니터 등의 그래픽 하드웨어를 추상화하여 관리
- **플랫폼 독립성**: Direct3D 버전(D3D11, D3D12)과 무관하게 공통 인터페이스 제공
- **디스플레이 관리**: 전체화면/창모드 전환, 해상도 변경, 다중 모니터 지원
- **리소스 공유**: 여러 그래픽 API 간 리소스 공유 가능
- **버퍼 관리**: 스왑체인을 통한 버퍼링 및 화면 출력

### 주요 DXGI 인터페이스

#### IDXGIFactory

**역할**: DXGI를 사용하기 위해 필요한 각종 인터페이스 생성의 진입점

#### 주요 기능
- **IDXGIAdapter 열거**: 시스템에 설치된 그래픽 카드 목록 조회
- **IDXGISwapChain 생성**: 화면 출력을 위한 스왑체인 생성
- **전체화면 상태 관리**: 전체화면 전환 및 관리
